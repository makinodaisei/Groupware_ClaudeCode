# UX全面リニューアル 設計書

**日付:** 2026-03-20
**対象:** `frontend/index.html`（フロントエンドのみ。バックエンドAPIは変更なし）
**方針:** フルリライト（Approach B）— 単一HTMLファイル構成を維持しつつ全面刷新

---

## 1. 設計の目的

現行UIの以下の問題を解消し、操作感を劇的に改善する。

| 現状の問題 | 解決策 |
|---|---|
| `prompt()` でISO-8601形式を手打ち | カレンダーピッカー + 5分刻みドロップダウン |
| `alert()` でフィードバック・ファイル一覧表示 | トースト通知 + スライドインパネル + ファイルグリッド |
| ナビが横並びボタンで画面幅に弱い | 縦サイドバーナビへ変更 |
| ダッシュボード統計が未完成（`-`表示） | 全APIを並列呼び出しし統計を完成させる |
| ページ遷移・操作フィードバックなし | フェードイン・スライドイン・スピナー追加 |

---

## 2. デザイン方針

- **トーン:** プロフェッショナル・落ち着き（Microsoft 365ライク）
- **プライマリカラー:** 濃紺 `#0f172a` / アクセント `#1d4ed8`（青）
- **背景:** `#f8fafc`（ライトグレー）
- **カード:** 白 `#ffffff` + `border: 1px solid #e2e8f0` + 軽いシャドウ
- **アニメーション:** 適度（フェード・スライド・ホバーエフェクト）

## 3. レイアウト構造

### 3-1. アプリシェル

```
┌─────────────────────────────────────────┐
│ TopBar (高さ 48px)                        │
│  [🏢 社内グループウェア]    [ユーザー名 🔵] │
├───────┬─────────────────────────────────┤
│ Side  │                                 │
│ bar   │  メインコンテンツエリア           │
│(52px) │                                 │
│  🏠   │                                 │
│  📅   │                                 │
│  🏢   │                                 │
│  📁   │                                 │
│  ─    │                                 │
│  ⚙️   │                                 │
└───────┴─────────────────────────────────┘
```

- TopBar: 固定、ユーザー名表示（ロールがadminなら管理バッジ）
- Sidebar: 固定、アイコンのみ（52px幅）。アクティブ状態は背景色変化
- メインエリア: スクロール可能、`max-width: 1200px` センタリング
- セクション切り替え: フェードイン（150ms）

### 3-2. ログイン画面

現行の中央配置ボックスを維持。改善点：
- 左半分にイメージ（濃紺グラデーション + アプリロゴ）
- 右半分にフォーム
- Enterキーでsubmit（既存ロジック維持）
- エラーメッセージをインラインで赤表示

## 4. 各セクション設計

### 4-1. ダッシュボード

**統計カード（4枚）:**
- 白背景、アイコン、数値、ラベル、トレンド表示
- 表示項目: 今月のスケジュール数・今日の施設予約数・文書フォルダ数・ユーザー数
- 実装: `Promise.all` で4つのAPIを並列呼び出し

**直近のスケジュール:**
- リスト形式（ドット + タイトル + 時刻）
- 最大5件表示

### 4-2. スケジュール（カレンダー）

**カレンダー表示:**
- 既存のグリッド形式を維持、スタイルを刷新
- 今日をハイライト、イベントのある日に青ドット表示
- カレンダーの日付セルをクリックするとその日付でドロワーを開く

**イベント追加ドロワー:**
- 画面右からスライドイン（transform translateX アニメーション 200ms）
- オーバーレイ（半透明）クリックで閉じる、ESCキーでも閉じる
- フォーム項目:
  - タイトル（テキスト入力）
  - 場所（テキスト入力）
  - 開始日（カスタムカレンダーポップアップ）+ 開始時刻（5分刻みセレクト: 00:00〜23:55）
  - 終了日（カスタムカレンダーポップアップ）+ 終了時刻（5分刻みセレクト）
  - 公開/非公開（トグル）
- 保存ボタン押下中はスピナー表示、完了後トースト通知

**カレンダーポップアップ:**
- インラインで実装（外部ライブラリなし）
- 入力フィールドクリックで下に展開
- 月ナビゲーション（◀▶）
- 選択日はハイライト

### 4-3. 施設予約

**施設一覧:**
- カード形式（施設名・収容人数・場所・説明）
- 「予約する」ボタンでドロワーを開く

**予約ドロワー:**
- スケジュールと同じ構造のスライドインパネル
- フォーム項目: 予約タイトル・開始日時・終了日時・メモ
- 重複エラー時はドロワー内にインラインエラー表示（`alert()` 不使用）

**空き状況タイムライン（当日表示）:**
- 各施設カードに当日の予約を横バーで表示（9:00〜18:00）
- 予約済みブロックをグレー、空き時間を緑で表示

### 4-4. 文書管理

**レイアウト:**
- 左側: フォルダツリー（固定幅 200px）
  - 階層表示（インデント）
  - クリックで選択、選択状態をハイライト
- 右側: ファイルエリア
  - 上部: パンくずナビゲーション（ホーム › フォルダ名 › …）
  - ファイルグリッド（3列）: アイコン + ファイル名 + サイズ
  - ファイルクリックでダウンロードURL取得 → 新タブで開く
  - 「＋追加」カードでファイルアップロードダイアログを開く

**ファイルアップロード:**
- ドラッグ&ドロップエリア または ファイル選択ボタン
- アップロード進捗をプログレスバーで表示
- 完了後トースト通知

**フォルダ作成:**
- ツリー上部の「＋」ボタン → インラインで名前入力

### 4-5. ユーザー管理（admin限定）

**ユーザーテーブル:**
- 現行のテーブル構成を維持しスタイル刷新
- ステータスバッジのデザイン改善

**ユーザー招待:**
- スライドインドロワー（`prompt()` 廃止）
- フォーム: メールアドレス・表示名・ロール選択（ラジオボタン）

## 5. 共通UIコンポーネント

### 5-1. ドロワー（Drawer）

```javascript
// 共通ドロワー管理
openDrawer(title, contentHTML)  // 右からスライドイン
closeDrawer()                   // ESC or オーバーレイクリック or ✕ボタン
```

### 5-2. トースト通知（Toast）

```javascript
showToast(message, type)  // type: 'success' | 'error' | 'info'
// 画面右下に表示、3秒後に自動消去
// フェードイン・フェードアウトアニメーション
```

### 5-3. カスタムカレンダーポップアップ

```javascript
// <input type="text"> の下に展開するインラインカレンダー
// 選択後は YYYY/MM/DD 形式で表示
// ISO-8601（YYYY-MM-DDThh:mm:ss+09:00）はAPIへの送信時に変換
```

### 5-4. 時刻セレクト（5分刻み）

```javascript
// <select> に 00:00〜23:55 を5分刻みで生成（288オプション）
generateTimeOptions()  // → ["00:00", "00:05", ..., "23:55"]
```

### 5-5. ローディング状態

- ボタン: 押下中はスピナーアイコン + disabled
- テーブル/リスト: スケルトンローディング（3行分のシマシマ）
- 統計カード: 数値がロード中は `—` 表示

## 6. アニメーション仕様

| 要素 | アニメーション | 時間 |
|---|---|---|
| セクション切り替え | opacity 0→1 + translateY 8px→0 | 150ms |
| ドロワー開く | translateX 100%→0 | 200ms ease-out |
| ドロワー閉じる | translateX 0→100% | 150ms ease-in |
| オーバーレイ | opacity 0→0.4 | 200ms |
| トースト表示 | translateY 100%→0 + opacity | 250ms |
| トースト消去 | opacity 1→0 | 300ms |
| ホバーエフェクト | background-color, box-shadow | 120ms |

## 7. 実装スコープ（変更しないもの）

- バックエンドAPI（エンドポイント・スキーマ・認証方式）
- Cognito認証ロジック（`login()` 関数の本体）
- `CONFIG` オブジェクト（apiEndpoint, userPoolId, clientId）
- デモモード（apiEndpointなしでの動作）

---

## 8. ファイル構成

変更は `frontend/index.html` 1ファイルのみ。
CSS・JavaScriptはすべてインライン（`<style>` + `<script>`）。
外部ライブラリは使用しない（CDNなし）。

---

## 9. 成功基準

- `prompt()` / `alert()` / `confirm()` を一切使用しない
- すべてのデータ入力がUIコンポーネント経由で完結する
- ISO-8601文字列のAPIへの変換はすべてJavaScript内で処理
- ダッシュボードの全統計値が正しく表示される
- ドロワー・トースト・カレンダーピッカーが動作する
