# UX全面リニューアル 設計書

**日付:** 2026-03-20
**対象:** `frontend/index.html`（フロントエンドのみ。バックエンドAPIは変更なし）
**方針:** フルリライト（Approach B）— 単一HTMLファイル構成を維持しつつ全面刷新

---

## 1. 設計の目的

現行UIの以下の問題を解消し、操作感を劇的に改善する。

| 現状の問題 | 解決策 |
|---|---|
| `prompt()` でISO-8601形式を手打ち | カレンダーピッカー + 5分刻みドロップダウン |
| `alert()` でフィードバック・ファイル一覧表示 | トースト通知 + スライドインパネル + ファイルグリッド |
| ナビが横並びボタンで画面幅に弱い | 縦サイドバーナビへ変更 |
| ダッシュボード統計が未完成（`-`表示） | 全APIを並列呼び出しし統計を完成させる |
| ページ遷移・操作フィードバックなし | フェードイン・スライドイン・スピナー追加 |

---

## 2. デザイン方針

- **トーン:** プロフェッショナル・落ち着き（Microsoft 365ライク）
- **プライマリカラー:** 濃紺 `#0f172a` / アクセント `#1d4ed8`（青）
- **背景:** `#f8fafc`（ライトグレー）
- **カード:** 白 `#ffffff` + `border: 1px solid #e2e8f0` + 軽いシャドウ
- **アニメーション:** 適度（フェード・スライド・ホバーエフェクト）

- TopBar: 固定、ユーザー名クリックでドロップダウンメニューを表示（「ログアウト」を含む）
- Sidebar: 固定、アイコンのみ（52px幅）。アクティブ状態は背景色変化。adminユーザーにはユーザー管理アイコン（👥）を追加表示
- メインエリア: スクロール可能、`max-width: 1200px` センタリング
- セクション切り替え: フェードイン（150ms）

### 3-2. ログイン画面

- 左半分: 濃紺グラデーション（`#0f172a`→`#1e3a5f`）+ アプリロゴ（中央配置）
- 右半分: フォーム（メールアドレス・パスワード・ログインボタン）
- Enterキーでsubmit（既存ロジック維持）
- エラーメッセージをインラインで赤表示
- **レスポンシブ:** 768px未満では左パネルを非表示にし、フォームのみ中央配置

## 4. 各セクション設計

### 4-1. ダッシュボード

**統計カード（4枚）:**
- 白背景、アイコン、数値、ラベル、サブ情報を表示
- 実装: `Promise.all` で下記4エンドポイントを並列呼び出し

| カード | エンドポイント | 取得フィールド |
|---|---|---|
| 今月のスケジュール数 | `GET /schedules?month={YYYY-MM}` | `count` |
| 今日の施設予約数 | `GET /facilities/{id}/reservations?date={YYYY-MM-DD}` を施設ごとに呼び出し合算 | `reservations.length` の合計 |
| 文書フォルダ数 | `GET /documents/folders` | `folders.length` |
| ユーザー数 | `GET /users` | `users.length` |

※ 施設予約数はまず `GET /facilities` で施設一覧を取得してから各施設の当日予約を並列取得する。エラー時は各カードに `—` を表示（他カードに影響しない）。

**直近のスケジュール:**
- リスト形式（ドット + タイトル + 時刻）
- 最大5件表示

### 4-2. スケジュール（カレンダー）

**カレンダー表示:**
- 既存のグリッド形式を維持、スタイルを刷新
- 今日をハイライト、イベントのある日に青ドット表示
- イベントチップ（既存イベント）クリックで編集ドロワーを開く
- 空白セルクリックでその日付を初期値として追加ドロワーを開く

**イベント追加/編集ドロワー:**
- 画面右からスライドイン（translateX アニメーション 200ms）
- オーバーレイ（半透明 `rgba(0,0,0,0.4)`）クリックで閉じる、ESCキーでも閉じる
- フォーム項目:
  - タイトル（テキスト入力、必須）
  - 場所（テキスト入力）
  - 開始日（カレンダーポップアップ）+ 開始時刻（5分刻みセレクト: 00:00〜23:55）
  - 終了日（カレンダーポップアップ）+ 終了時刻（5分刻みセレクト）
  - 公開/非公開（トグル）
- 保存ボタン押下中はスピナー表示、完了後トースト通知（`showToast`）
- **編集モード:** 「削除」ボタンを追加表示。削除確認はドロワー内の確認UI（「本当に削除しますか？」+ 「削除する」/「キャンセル」ボタン）で行い、`confirm()` は使用しない
- **バリデーション:** タイトル未入力時、終了時刻が開始時刻より前の場合はドロワー内にインラインエラー表示

**ドロワーAPI:**
```javascript
openDrawer(title, contentHTML, onSubmit)
// title: ドロワーヘッダーのタイトル文字列
// contentHTML: フォームのHTML文字列
// onSubmit: 保存ボタン押下時のコールバック (formData) => Promise<void>
//   - Promiseがresolveするとドロワーを閉じてトーストを表示
//   - Promiseがrejectするとドロワー内のエラー領域にメッセージを表示
closeDrawer()
```

**カレンダーポップアップ:**
- インラインで実装（外部ライブラリなし）
- 入力フィールドクリックで `position: fixed` で展開（z-index: ドロワーより上）
- 外側クリックで閉じる（ドロワーのオーバーレイより上のz-indexを持つためオーバーレイには伝播しない）
- 月ナビゲーション（◀▶）、選択日はハイライト
- ポップアップは `<body>` 直下に追加し、ドロワーの `overflow` の影響を受けない

### 4-3. 施設予約

**施設一覧:**
- カード形式（施設名・収容人数・場所・説明）
- 「予約する」ボタンでドロワーを開く

**予約ドロワー:**
- `openDrawer()` を使用
- フォーム項目: 予約タイトル・開始日時・終了日時・メモ
- 重複エラー（APIレスポンス `error: "CONFLICT"`）時はドロワー内のエラー領域にインライン表示（`alert()` 不使用）

**空き状況タイムライン（当日表示）:**
- 各施設カード内に表示する横軸タイムライン（9:00〜18:00）
- データ取得: `GET /facilities/{facilityId}/reservations?date={today}` を各施設ごとに並列呼び出し
- 予約済みブロック: グレー（`#94a3b8`）、空き時間: 緑（`#bbf7d0`）
- タイムラインの幅 = 9時間（540分）= 100%
- `left = (startMin - 540) / 540 * 100%`（startMin: 開始時刻の0時からの分数）
- `width = (endMin - startMin) / 540 * 100%`
- 9:00より前に開始する予約: `left` を `0%` にクランプし、`width` も `endMin - 540` ベースで再計算
- 18:00以降の予約（startMin >= 1080）は非表示。18:00をまたぐ予約は18:00でクリップ（`min(endMin, 1080)`）

### 4-4. 文書管理

**レイアウト:**
- 左側: フォルダツリー（固定幅 200px）
- 右側: ファイルエリア（パンくず + ファイルグリッド）

**フォルダツリー構築:**
- `GET /documents/folders` はフラットなリストを返す（`path` フィールドが `/` 区切りの階層パス）
- クライアント側で `path` を解析してネスト構造を構築する
- 例: `path = "/全社共有/議事録"` → 「全社共有」の子として「議事録」をインデント表示
- クリックで選択（ハイライト）しファイルエリアを更新

**サブフォルダ作成:**
- ツリー上部の「＋」ボタン → インラインで名前入力フィールドを表示
- `POST /documents/folders` ボディ: `{ name, parentFolderId, parentPath }` （選択中フォルダをparentとして送信）

**ファイルエリア:**
- 上部: パンくずナビゲーション（ホーム › フォルダ名 › … ）。各クラムはクリック可能でそのフォルダを選択
- ファイルグリッド（3列）: アイコン（拡張子に応じて絵文字を切り替え） + ファイル名 + サイズ
- ファイルクリック: `GET /documents/folders/{folderId}/files/{fileId}/download-url` でURLを取得 → 新タブで開く
- 「＋追加」ドロップゾーン: ドラッグ&ドロップ または クリックでファイル選択

**ファイルアップロード:**
1. `POST /documents/folders/{folderId}/files/upload-url` でPresigned URLを取得
2. `XMLHttpRequest` + `upload.onprogress` でS3へPUT（`fetch` APIはアップロード進捗を取得できないため使用しない）
3. プログレスバーで進捗表示（0〜100%）
4. 完了後トースト通知

### 4-5. ユーザー管理（admin限定）

**ユーザーテーブル:**
- 現行のテーブル構成を維持しスタイル刷新
- ステータスバッジのデザイン改善

**ユーザー招待:**
- `openDrawer()` を使用（`prompt()` 廃止）
- フォーム: メールアドレス・表示名・ロール選択（ラジオボタン: user / admin）

## 5. 共通UIコンポーネント

### 5-1. ドロワー（Drawer）

```javascript
openDrawer(title, contentHTML, onSubmit)
// title: ドロワーヘッダーの文字列
// contentHTML: フォームのHTML文字列（保存ボタンは不要、ドロワーが自動生成）
// onSubmit: async (formData: Object) => void
//   - resolve → closeDrawer() + showToast('保存しました', 'success')
//   - reject(message) → ドロワー内 #drawer-error にメッセージを表示

closeDrawer()
// ESC or オーバーレイクリック or ✕ボタン or onSubmit resolve 時に呼ばれる
```

**ドロワーDOM構造:**
```html
<div id="drawer-overlay">          <!-- position:fixed, inset:0, z-index:100 -->
  <div id="drawer-panel">          <!-- position:fixed, right:0, top:0, height:100%, width:480px, z-index:101 -->
    <div id="drawer-header">...</div>
    <div id="drawer-body">         <!-- contentHTMLがここに挿入される -->
      <div id="drawer-error"></div><!-- バリデーション/APIエラー表示 -->
      <!-- フォームコンテンツ -->
    </div>
    <div id="drawer-footer">
      <button id="drawer-cancel">キャンセル</button>
      <button id="drawer-submit">保存</button>  <!-- スピナー付き -->
    </div>
  </div>
</div>
```

## 6. アニメーション仕様

| 要素 | アニメーション | 時間 |
|---|---|---|
| セクション切り替え | opacity 0→1 + translateY 8px→0 | 150ms |
| ドロワー開く | translateX 100%→0 | 200ms ease-out |
| ドロワー閉じる | translateX 0→100% | 150ms ease-in |
| オーバーレイ | opacity 0→0.4 | 200ms |
| トースト表示 | translateY 100%→0 + opacity | 250ms |
| トースト消去 | opacity 1→0 | 300ms |
| ホバーエフェクト | background-color, box-shadow | 120ms |
| スケルトン | shimmer (background-position) | 1.5s infinite |

## 9. 成功基準

- `prompt()` / `alert()` / `confirm()` をソース中に1件も使用しない
- すべてのデータ入力がUIコンポーネント（ドロワー・カレンダーピッカー・時刻セレクト）経由で完結する
- ISO-8601文字列の生成はJavaScript内のみで行い、ユーザーに文字列入力を求めない
- ダッシュボードの全4統計値がAPIから正しく取得・表示される
- ESCキーを押すと開いているドロワーが閉じる
- トースト通知は表示から3秒後に自動消去される
- カレンダーピッカーで日付を選択するとinput要素に `YYYY/MM/DD` 形式で入力される
- ファイルアップロード中にプログレスバーが表示される
- 既存イベントをクリックすると編集ドロワーが開き、削除できる
- デモモード（CONFIG.apiEndpointがデフォルト値以外に設定されていない状態）でもUIが壊れない
